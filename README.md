Compréhension des principes SOLID

Les principes SOLID sont cinq règles de conception orientée objet qui visent à rendre le code plus "lisible", "maintenable" et "extensible".

- "SRP (Single Responsibility Principle) ": Une classe ne doit avoir qu’une seule raison de changer, c’est-à-dire une seule responsabilité.
- "OCP (Open/Closed Principle)" : Les entités logicielles doivent être ouvertes à l’extension, mais fermées à la modification.
- "LSP (Liskov Substitution Principle)" : Les objets d’un sous-type doivent pouvoir remplacer ceux de leur super-type sans altérer le comportement du programme.
- "ISP (Interface Segregation Principle)" : Il vaut mieux plusieurs interfaces spécifiques que une interface générale imposant des méthodes inutiles.
- "DIP (Dependency Inversion Principle)" : Dépendre des abstractions (interfaces), pas des implémentations concrètes.

Ce devoir illustre chaque principe avec un exemple "avant"(violant le principe) et "après" (corrigé).
